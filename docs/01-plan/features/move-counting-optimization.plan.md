# 계획서: 이동 횟수(Moves) 카운팅 로직 최적화

## 1. 개요
현재 점을 터치하기만 해도 올라가거나, 같은 색상을 수정할 때마다 증가하는 이동 횟수(Moves) 산정 방식을 개선합니다. 실제 파이프가 그려졌을 때만 카운트하고, 다른 색상의 파이프를 그리기 전까지는 동일한 색상의 수정 작업을 하나의 이동으로 간주하도록 로직을 변경합니다.

## 2. 핵심 수정 사항
- [ ] **카운트 시점 변경**: `path-started` (터치 시점)가 아닌, 파이프의 길이가 2개 이상의 셀을 점유하는 시점(`length >= 2`)에 카운트 시작.
- [ ] **색상 기반 중복 방지**:
    - 마지막으로 이동 횟수를 증가시킨 색상을 기억 (`lastMovedColor`).
    - 현재 그리는 색상이 `lastMovedColor`와 다를 때만 새로운 이동으로 간주하여 `moves` 증가.
    - 단, 파이프를 완전히 지웠다가 다시 그리는 등의 예외 상황에서도 논리적으로 타당한 카운팅 유지.
- [ ] **데이터 연동**: `MainScene`과 `PathManager` 간의 이벤트를 세분화하여 정확한 시점에 HUD 업데이트.

## 3. 기술 스택 및 전략
- **이벤트 시스템**: `path-started` 대신 `move-incremented`와 같은 전용 이벤트를 정의하거나, `PathManager` 내부에서 카운팅 조건을 판별.
- **상태 관리**: `MainScene`에서 `lastMovedColor` 변수를 추가하여 상태 추적.

## 4. 상세 작업 목록
- [ ] `PathManager.js`의 `extendPath` 메서드 수정: 파이프 길이가 2가 되는 순간을 감지하는 로직 추가.
- [ ] `MainScene.js`의 이동 횟수 증가 로직 수정:
    - `lastMovedColor` 변수 도입.
    - 새로운 색상의 파이프가 실질적으로 그려질 때만 `moves++`.
- [ ] 레벨 초기화(Reset) 시 `lastMovedColor` 초기화.
- [ ] 다양한 사용자 조작 케이스(터치만 하기, 같은 색상 반복 수정, 다른 색상 교차 그리기 등) 테스트.

## 5. 성공 지표
- 점을 터치만 하고 드래그하지 않았을 때는 이동 횟수가 변하지 않음.
- 같은 색상의 파이프를 계속 수정하거나 되돌려도(Backtrack) 이동 횟수가 추가로 증가하지 않음.
- 다른 색상의 파이프를 그리기 시작하여 실제 길이가 발생했을 때만 이동 횟수가 정확히 1 증가함.
