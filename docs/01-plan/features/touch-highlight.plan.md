# 계획서: 터치 하이라이트 효과 추가 (Touch Highlight Effect)

## 1. 개요
사용자가 점을 터치하고 드래그할 때, 현재 선택된 색상의 커다란 반투명 원이 터치 위치를 따라다니게 하여 조작 중인 위치와 색상을 명확하게 시각화합니다. (`touch_highlight.png` 이미지 참조)

## 2. 핵심 디자인 요소 (이미지 분석 반영)
- [ ] **하이라이트 원 (Highlight Circle)**:
    - **크기**: 점(Dot) 크기의 약 2배 혹은 셀 크기에 비례하는 크기.
    - **색상**: 현재 드래그 중인 파이프/점의 색상과 동일하게 적용.
    - **투명도**: 약 0.3 ~ 0.4 Alpha 적용 (반투명).
- [ ] **동작 로직**:
    - **표시**: 점을 터치하여 드래그가 시작되는 순간 나타남.
    - **이동**: 마우스/터치 포인터의 좌표(`pointer.x, pointer.y`)를 실시간으로 추적.
    - **숨김**: 터치를 떼는 순간(`pointerup`) 화면에서 사라짐.

## 3. 기술 스택 및 구현 전략
- **Phaser 3 Graphics**: 실시간으로 위치와 색상을 변경하기 용이한 `Graphics` 객체 또는 단일 `Circle` 게임 오브젝트 활용.
- **PathManager 연동**: 현재 선택된 색상 정보를 `PathManager`로부터 가져와 적용.
- **MainScene 업데이트**: `pointermove` 이벤트에서 하이라이트의 좌표를 갱신.

## 4. 상세 작업 목록
- [ ] **UIManager 또는 MainScene 확장**: 하이라이트 원 객체 생성 및 초기 상태(숨김) 설정.
- [ ] **MainScene.js 수정**:
    - `setupInputs()`의 `pointerdown` 시점에 하이라이트 활성화 및 색상 설정.
    - `pointermove` 시점에 하이라이트 위치를 `pointer` 위치로 갱신.
    - `pointerup` 시점에 하이라이트 비활성화.
- [ ] **시각적 최적화**: 선(Line)이나 점(Dot)보다 위에 그려지도록 `depth` 설정 (예: 15).

## 5. 성공 지표
- `touch_highlight.png` 이미지와 시각적으로 동일한 느낌의 하이라이트가 표시됨.
- 드래그 중 지연 없이 포인터를 부드럽게 따라다님.
- 터치 종료 시 잔상 없이 즉시 사라짐.
