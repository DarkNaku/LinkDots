# 계획서: 경로 곡선 라운딩 처리 (Smooth Path Rendering) - 수정안

## 1. 개요
Phaser 3 Graphics의 기본 `lineStyle`만으로는 구현하기 어려운 부드러운 꺾임 효과를 위해, 선의 연결 마디(Joints)마다 원형 그래픽을 추가하여 시각적으로 완벽하게 둥근 경로를 구현합니다.

## 2. 버그 원인 분석
- **Phaser 렌더러 제한**: 특정 Phaser 버전이나 WebGL 환경에서 `lineJoin: 'round'` 속성이 `lineTo` 연속 호출 시 기대만큼 부드럽게 표현되지 않거나 무시될 수 있음.
- **좌표 불일치**: 직선 연결 방식은 90도 전환 시 바깥쪽 모서리가 항상 날카롭게 남음.

## 3. 해결 전략 (Hybrid Rendering)
- **선(Line)**: 기존처럼 `moveTo`, `lineTo`를 사용하여 마디 사이를 잇습니다.
- **마디(Joints)**: 모든 셀 좌표(마디) 위치에 선의 두께(`thickness`)를 지름으로 하는 채워진 원(`fillCircle`)을 그립니다.
- **효과**: 직선과 원이 겹치면서 꺾이는 부분이 자연스럽게 밖으로 볼록하고 둥글게 처리되어 완벽한 '파이프' 형태가 완성됩니다.

## 4. 상세 작업 목록
- [ ] `PathManager.js`의 `draw()` 메서드 로직 전면 개편.
- [ ] 경로를 구성하는 각 `Cell`마다 `fillCircle`을 먼저 그려 마디를 생성.
- [ ] 마디 사이를 `lineStyle`이 적용된 직선으로 연결.
- [ ] 선의 두께와 원의 지름을 동일하게 설정 (`cells[0].size * 0.45`).

## 5. 성공 지표
- 선이 90도로 꺾이는 구간의 바깥쪽이 부드러운 곡선으로 표시됨.
- 선의 시작점과 끝점도 원형으로 자연스럽게 마감됨.
- 드래그 중에도 끊김 없이 부드러운 경로가 실시간으로 렌더링됨.
