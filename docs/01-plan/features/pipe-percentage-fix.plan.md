# 계획서: 파이프 점유율 계산 버그 수정 (Pipe Percentage Calculation Fix)

## 1. 개요
게임 시작 시 또는 파이프가 없는 상태에서 파이프 점유율(Pipe %)이 0%가 되지 않는 현상을 수정합니다. 현재 로직이 '파이프가 지나가는 칸'뿐만 아니라 '점이 배치된 칸'까지 모두 점유된 것으로 계산하고 있는 것이 원인입니다.

## 2. 버그 원인 분석
- **`GridManager.js`의 `getFillPercentage()`**:
    ```javascript
    if (this.cells[y][x].isOccupied || this.cells[y][x].dotColor) {
        occupied++;
    }
    ```
    위 코드에서 `dotColor`가 있는 칸(시작/종료 점)을 항상 점유된 것으로 처리하고 있습니다. 이로 인해 5x5 그리드에서 5쌍(10개)의 점이 있다면 시작부터 40%의 점유율이 표시되는 문제가 발생합니다.

## 3. 해결 전략
- **점유 기준 변경**: 점유율 계산 시 `dotColor` 유무와 관계없이, 실제 파이프가 그려진 상태를 나타내는 `isOccupied` 값만 기준으로 계산하도록 수정합니다.
- **파이프 연결 로직 확인**: `PathManager`에서 파이프가 시작될 때 해당 점(Dot)의 `isOccupied`를 `true`로 설정하고 있으므로, 파이프가 그려진 만큼만 정확히 점유율에 반영될 것입니다.

## 4. 상세 작업 목록
- [ ] `src/classes/GridManager.js` 내 `getFillPercentage()` 메서드 수정.
- [ ] `isAllFilled()` 메서드 검토: 승리 조건에서는 모든 칸이 파이프로 채워져야 하므로, 이 로직도 `isOccupied` 중심으로 동작하는지 확인.
- [ ] 수정 후 게임 초기 상태 및 파이프 삭제 시 점유율이 0%로 떨어지는지 검증.

## 5. 성공 지표
- 레벨 시작 시 파이프 점유율이 정확히 **0%**로 표시됨.
- 파이프를 그릴 때마다 점유율이 실시간으로 증가함.
- 모든 파이프를 지우면 다시 **0%**가 됨.
- 모든 칸을 파이프로 채웠을 때만 **100%**가 됨.
