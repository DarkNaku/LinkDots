# 계획서: 화면 비율 최적화 (9:16 Aspect Ratio)

## 1. 개요
현재 600x800(3:4)으로 설정된 게임 해상도를 모바일 표준 비율인 9:16(예: 720x1280 또는 540x960)으로 변경하고, 모든 Scene의 UI 요소들이 새로운 비율에서도 조화롭게 배치되도록 조정합니다.

## 2. 핵심 수정 사항
- [ ] **Phaser 엔진 설정 변경**: `src/main.js`의 `width`, `height`를 9:16 비율로 수정 (추천: 720x1280).
- [ ] **레이아웃 재배치**:
    - **MainScene**: 그리드 크기와 위치를 세로로 긴 화면에 맞춰 조정.
    - **TitleScene**: 로고와 메뉴 리스트의 간격 조정.
    - **PackSelect/LevelSelect**: 리스트와 그리드의 수직 여백 및 페이지네이션 위치 조정.
- [ ] **UIManager 대응**: 하드코딩된 좌표값(예: 300, 800 등)을 화면 크기 기반(`this.width`, `this.height`)의 상대 좌표로 전환하여 유연성 확보.

## 3. 기술 스택 및 전략
- **해상도**: 720x1280 (HD 9:16 표준).
- **Scale Mode**: `Phaser.Scale.FIT` 유지 (다양한 디바이스 대응).
- **좌표 계산**: 중앙 정렬(`width / 2`) 및 화면 비율에 따른 퍼센트 기반 배치 유도.

## 4. 상세 작업 목록
- [ ] `src/main.js`에서 해상도를 720x1280으로 변경.
- [ ] `UIManager.js` 내의 모든 정적 좌표값(예: 300)을 `this.width / 2` 등으로 동적 변환.
- [ ] `MainScene.js`의 그리드 크기(`gridSize`)를 새로운 너비에 맞춰 조정 (예: 400 -> 500).
- [ ] 각 Scene별로 요소들이 겹치거나 너무 벌어지지 않는지 전수 점검 및 수정.
- [ ] `TitleScene` 배경 효과(도트 생성 범위)를 확장된 해상도에 맞춰 수정.

## 5. 성공 지표
- 게임 화면이 상하로 길어져 최신 스마트폰 화면을 꽉 채움.
- 모든 UI 요소(타이틀, 버튼, 그리드)가 화면 중앙에 안정적으로 배치됨.
- 씬 전환 및 입력 처리(좌표 인식)가 변경된 해상도에서도 정확히 작동함.
