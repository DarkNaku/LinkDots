# 계획서: 레벨 시스템 확장 (LinkDots)

## 1. 개요
현재 `MainScene` 내부에 하드코딩된 레벨 데이터를 외부 JSON 파일로 분리하고, 다수의 레벨을 순차적으로 플레이하거나 선택할 수 있는 시스템을 구축합니다.

## 2. 핵심 기능
- [ ] **외부 데이터 분리**: 레벨 데이터를 `assets/levels/levels.json` 파일로 이동.
- [ ] **레벨 로더 구현**: Phaser의 `load.json`을 사용하여 데이터를 동적으로 로드.
- [ ] **레벨 관리자(LevelManager) 추가**:
    - 현재 레벨 추적.
    - 다음 레벨로 전환 로직.
    - 레벨 팩(5x5, 6x6 등) 구분 지원.
- [ ] **진행 시스템**:
    - 레벨 클리어 시 'Next Level' 버튼 표시.
    - 마지막 레벨 클리어 시 축하 메시지 및 처음으로 가기.
- [ ] **UI 확장**:
    - HUD에 현재 레벨 번호 표시 방식 개선.
    - 간단한 레벨 선택 창 (선택 사항).

## 3. 기술 스택
- **데이터 형식**: JSON
- **에셋 관리**: Phaser Loader (Scene.load)
- **로직 처리**: Singleton 패턴 또는 Scene 프로퍼티를 활용한 레벨 관리

## 4. 상세 작업 목록
- [ ] 레벨 데이터를 구조화하여 `assets/levels/levels.json` 생성.
- [ ] `MainScene`의 `preload()` 단계에서 JSON 파일 로드 로직 추가.
- [ ] `MainScene`의 `create()` 시점에 JSON 데이터를 파싱하여 그리드 초기화.
- [ ] 승리 팝업에 'NEXT LEVEL' 버튼 추가 및 클릭 이벤트 구현.
- [ ] 레벨 전환 시 기존 그리드와 경로를 완벽히 정리(Cleanup)하는 로직 강화.

## 5. 성공 지표
- 외부 JSON 파일 수정만으로 게임 레벨을 추가/변경할 수 있음.
- 'Next Level' 클릭 시 다음 레벨이 오류 없이 로드되고 플레이 가능함.
- 모든 레벨을 순차적으로 플레이하는 흐름이 매끄러움.
